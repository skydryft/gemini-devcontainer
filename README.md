# Gemini Devcontainer — Rails

A templated devcontainer bootstrapper for Ruby on Rails projects using Gemini CLI as the primary AI development assistant. Provides a pre-configured development environment with Rails, PostgreSQL, and our custom [gemini-cli-gt](../gemini-cli-gt/) fork running in shell-only mode.

## What's Included

- **Ruby** (latest stable) with Rails and Bundler pre-installed
- **PostgreSQL** running locally with dev credentials ready to go
- **Gemini CLI GT** built from source with shell-only mode enabled
- **GEMINI.md** template with Rails-specific workflow guidance
- Credential persistence across container rebuilds via shared Docker volumes

## Quick Start

1. Copy this directory as the base for a new Rails project
2. Open in VS Code / Codespaces — the devcontainer builds automatically
3. Create your Rails app:
   ```bash
   rails new myapp --database=postgresql
   cd myapp
   rails db:create
   rails server
   ```
4. Run `gemini` in the terminal to start coding with AI assistance
5. Update `GEMINI.md` with your app's context and requirements

## Database

PostgreSQL is installed and configured locally inside the container.

| Setting | Value |
|---|---|
| Host | localhost |
| Port | 5432 |
| Username | rails |
| Password | rails |
| Dev Database | app_development |
| Test Database | app_test |
| `DATABASE_URL` | Set automatically in the environment |

The `database.yml` generated by `rails new --database=postgresql` will work out of the box when configured with these credentials.

## Shell-Only Mode

This template ships with shell-only mode enabled by default. Instead of giving the model 25+ specialized tools to choose from, it gets 4:

| Tool | Purpose |
|---|---|
| `run_shell_command` | All file operations, builds, tests — everything goes through bash |
| `ask_user` | Ask clarifying questions back to the user |
| `update_topic` | Conversation topic metadata |
| `save_memory` | Persist facts across sessions |

The model uses standard Unix commands (`grep`, `find`, `cat`, `sed`, etc.) for file operations. This plays to Gemini's training strengths and eliminates tool selection overhead.

See [shell-only mode research](../docs/research/gemini-cli/shell-only-mode.md) for the rationale and test results.

## Configuration

### `.gemini/settings.json`

```json
{
  "tools": {
    "core": ["run_shell_command", "ask_user", "update_topic", "save_memory"]
  }
}
```

To revert to the full tool set, remove the `tools.core` key entirely.

### `GEMINI.md`

The project context template. Fill in the `<project_context>` section with your app description and update commands once established. Keep this file lean — context length directly impacts Gemini flash response times.

### Credential Persistence

Credentials are stored in a shared Docker named volume (`gemini-config`):

- First time: run `gemini` and authenticate via `/auth`
- Credentials auto-restore on every container start or rebuild
- Shared across all devcontainer projects using this bootstrapper

## Project Structure

```
.devcontainer/
  devcontainer.json    Container config (Ruby, Node.js, PostgreSQL)
  Dockerfile           Base image, system deps, PostgreSQL setup
  post-create.sh       First-run setup (gemini-cli-gt build, Rails install, DB init)
  post-start.sh        PostgreSQL start, auth status on every start
.gemini/
  settings.json        Gemini CLI settings (shell-only mode)
  session-notes/       Session summaries (written by the model)
  tracker/tasks/       Task tracking (managed by the model)
GEMINI.md              Project context template (Rails-specific)
```

## Customizing

The environment is ready for Rails development out of the box. To add additional tools or services:

- Add dev container features to `devcontainer.json` (e.g., Redis, Elasticsearch)
- Add system packages to the `Dockerfile`
- Add setup steps to `post-create.sh`

After changes, rebuild the container (`Cmd/Ctrl+Shift+P` > "Rebuild Container").
